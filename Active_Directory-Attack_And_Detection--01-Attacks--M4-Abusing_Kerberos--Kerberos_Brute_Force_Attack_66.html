Kerberos Brute Force Attack

  1. 1.

    1. Hypothesis & Concept

Hypothesis: We suspected that the shinobi.local domain might have weak password policies or predictable usernames that could be guessed via the Kerberos service.

Concept:

  • Enumeration: The Kerberos service responds differently to TGT requests for valid users versus invalid users. We can exploit these error codes (KDC_ERR_C_PRINCIPAL_UNKNOWN vs KDC_ERR_PREAUTH_REQUIRED) to build a list of valid domain users.

  • Brute Force: Once valid users are known, we can attempt to authenticate by sending AS-REQ packets encrypted with guessed passwords. If the KDC returns a TGT, the password is correct.

  1. 2.

    2. Phase 1: User Enumeration (Nmap)

Objective: Identify valid user accounts in the domain without having any initial credentials. Tool: Nmap (NSE Script: krb5-enum-users) Command:

nmap -p 88 --script krb5-enum-users --script-args krb5-enum-users.realm="shinobi.local",userdb=users.txt 192.168.56.10

Findings:

  • The script tested a list of potential usernames (users.txt) against the Domain Controller.

  • It successfully identified 8 valid principals, including nezuko_demon18, svc_deathnote, naruto_uzumaki07, and tanjiro_kamado21.

  • This output provides a verified target list for the next phase.

  1. 3.

    3. Phase 2: Password Brute Force (Kerbrute)

Objective: Perform a password spraying/brute force attack against the valid users identified in Phase 1 to compromise an account. Tool: kerbrute.py (Python implementation)

Command:

python3 kerbrute.py -dc-ip 192.168.56.10 -domain shinobi.local -users discovered_users.txt -passwords password.txt -outputfile shinobi_user_password.txt

Findings:

  • The tool iterated through the provided wordlist (password.txt) against the valid users.

  • Success: It found a valid credential pair:
  • User: tanjiro_kamado21@shinobi.local

  • Password: Hinokami@456

  • The tool successfully retrieved and saved the Ticket Granting Ticket (TGT) in a credential cache (.ccache) file, allowing for immediate reuse (Pass-the-Ticket).

  1. 4.

    4. Impact

By exploiting the Kerberos interface, we successfully enumerated domain users and compromised a user account (tanjiro_kamado21). This valid account can now be used as a foothold for further enumeration (BloodHound), lateral movement, or more advanced attacks like Kerberoasting.
