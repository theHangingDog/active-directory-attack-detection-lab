BloodHound


Post-Exploitation Enumeration (BloodHound)

  1. 1.

    1. Overview

BloodHound was utilized to visualize Active Directory relationships using graph theory. This allows for the identification of complex attack paths, hidden administrative relationships, and privilege escalation vectors that are invisible to standard CLI tools like net.exe.

  1. 2.

    2. Data Collection (Ingestor)

Objective: Stealthily gather domain structure data (Users, Groups, Computers, Sessions) from a Linux attacker machine without joining the domain.

  • Tool: bloodhound-python

  • Command:

bloodhound-python -u

'goku_saiyan88'

-p

'UltraInstinct!9'

 -ns 192.168.56.10 -d shinobi.local -c All

  • Findings:
  • Successfully connected to the LDAP server (dc01.shinobi.local).

  • Enumerated 22 Users, 55 Groups, and 2 Computers.

  • This collection phase confirms that any compromised user credentials (even low-level ones) allow for a complete mapping of the AD environment.

  1. 3.

    3. Data Processing & Import

Objective: Load the collected JSON data into the Neo4j database for visualization.

  • Action: Uploaded the generated JSON files (computers, domains, groups, users) into the BloodHound GUI.

  • Observation: The distinct JSON files confirm that the ingestor successfully categorized the raw LDAP data into ingestible graph nodes.

  1. 4.

    4. Visualizing Domain Privilege (Group Mapping)

Objective: Identify the "Keys to the Kingdom"—users who are members of the Domain Admins group.

  • Cypher Query:

Cypher

MATCH p = (t:Group)<-[:MemberOf*1..]-(a)

WHERE (a:User or a:Computer) and t.objectid ENDS WITH '-512'

RETURN p

  • Analysis:
  • Identified Administrator, madara_uchiha91, gojo_satoru69, and itachi_anbu01 as members of Domain Admins.

  • This visualization reveals the high-value targets an attacker would prioritize for credential theft or phishing.

  1. 5.

    5. Attack Path Analysis (Shortest Path)

Objective: Determine the most efficient "Kill Chain" to compromise the entire domain starting from any node.

  • Query: Shortest Paths to High Value Targets

  • Analysis:
  • The graph reveals a complex chain of permissions including AddKeyCredentialLink, GenericAll, and WriteDacl.

  • This confirms that compromising a seemingly unrelated user (e.g., naruto_uzumaki07) could lead to Domain Admin compromise via a chain of delegation and ACL abuse.

  1. 6.

    6. Identifying Vulnerable Service Accounts (Kerberoasting)

Objective: Locate accounts that are vulnerable to Kerberoasting (Module 4) to extract service ticket hashes for offline cracking.

  • Query: List all Kerberoastable Users (Accounts with Service Principal Names set).

  • Cypher Query Used:

Cypher

MATCH (u:User) WHERE u.hasspn=true AND u.enabled=true RETURN u

  • Findings:
  • Identified 3 targets: svc_deathnote, svc_sharingan, and svc_saiyan.

  • These accounts act as the "low hanging fruit" for lateral movement since their passwords can often be cracked offline.

Detection Note (Blue Team Perspective)

  • Large LDAP Queries: Monitoring for massive LDAP queries (Event ID 1644) from a single source IP often indicates BloodHound activity.

  • Honeytokens: Creating a fake, attractive user account (e.g., svc_backup) and monitoring it for "Shortest Path" queries or TGS requests can alert defenders to active reconnaissance.
