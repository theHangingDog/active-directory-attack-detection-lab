nxc ldap


Post-Exploitation Enumeration (NetExec LDAP)

  1. 1.

    1. Overview

  • Tool: NetExec
  • Protocol: LDAP (Lightweight Directory Access Protocol)
  • Credential Used: kaneki_ghoul25
  • Objective: Having compromised a valid domain user, we pivoted from noisy SMB scanning to Authenticated LDAP Enumeration. LDAP allows for deep queries of the Active Directory database (NTDS) to uncover configuration flaws, privilege escalation vectors, and hidden attributes that unauthenticated scans miss.

  1. 2.

    2. Domain SID Retrieval (Golden Ticket Prep)

Hypothesis: To prepare for future persistence attacks (Module 9: Golden Ticket), we need the Domain SID. This identifier is constant across the domain and is a required component for forging Kerberos tickets.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' --get-sid

  • Findings:
  • Domain SID: S-1-5-21-4057973084-1432748986-1156288224

  • Action: This SID has been noted for use with ticketer.py in later stages.

  1. 3.

    3. Password Policy Auditing

Hypothesis: Before attempting password spraying or brute-force attacks against other users, we must identify the Account Lockout Threshold. If the threshold is 0 (None), we can brute-force aggressively without risk of causing a Denial of Service.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' --pass-pol

  • Findings:
  • Account Lockout Threshold: 0 (Disabled).

  • Minimum Password Length: 7 characters.

  • Complexity: Enabled.

  • Strategic Value: The absence of a lockout threshold confirms the domain is highly vulnerable to brute-force attacks.

  1. 4.

    4. Comprehensive User Enumeration

Hypothesis: We need to map the entire "Human Attack Surface" to identify naming conventions and potential high-value targets that were not found during initial reconnaissance.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' --users

  • Findings:
  • Successfully dumped the entire directory.

  • Re-confirmed the description leak for kaneki_ghoul25.

  • Identified service accounts: svc_saiyan, svc_sharingan, svc_deathnote.

  1. 5.

    5. Unconstrained Delegation Discovery

Hypothesis: We are looking for computers or users trusted for Unconstrained Delegation. If we compromise such an object, we can wait for a Domain Admin to connect to it and steal their TGT from memory.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' --trusted-for-delegation

  • Findings:
  • Target Found: DC01$ (Domain Controller).

  • Analysis: While typical for DCs, finding any other workstation here would represent a critical misconfiguration.

  1. 6.

    6. RBCD Prerequisite Check (Machine Account Quota)

Hypothesis: We need to check the MachineAccountQuota (MAQ) attribute. If this value is > 0, any user (including our compromised kaneki) can create new computer accounts. This is the primary prerequisite for Resource-Based Constrained Delegation (RBCD) attacks.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' -M maq

  • Findings:
  • MachineAccountQuota: 10.

  • Conclusion: The domain is vulnerable. We can generate a new computer account to facilitate privilege escalation attacks.

  1. 7.

    7. High-Value Target Identification (AdminCount)

Hypothesis: We are searching for users with the adminCount=1 attribute. These users are protected by the AdminSDHolder process, indicating they are (or were) members of a privileged group like Domain Admins, Backup Operators, or Account Operators.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' --admin-count

  • Findings:
  • Validated our target list: Administrator, itachi_anbu01, krbtgt, gojo_satoru69, madara_uchiha91.

  • This list represents the "Tier 0" assets in the environment.

  1. 8.

    8. LAPS (Local Admin Password Solution) Audit

Hypothesis: If LAPS is deployed but permissions are misconfigured, a standard user might be able to read the local administrator passwords of workstations in cleartext.

  • Command: nxc ldap 192.168.56.10 -u kaneki_ghoul25 -p 'TokyoGhoul#1' -M laps

  • Findings:
  • Result: No results found with ms-Mcs-AdmPwd.

  • Conclusion: Either LAPS is not installed, or the permissions are correctly secured against our user.
