User Owns Generic Write Permission On Another User


Detection & Telemetry (Targeted Kerberoasting)

  1. 1.

    1. Overview

Attack Vector: The attacker (tanjiro_kamado21) abused GenericWrite permissions on the user sasuke_uchiha99. They injected a Service Principal Name (SPN) onto the victim account to facilitate a Targeted Kerberoasting attack, allowing them to request a crackable TGS ticket.

Key Log Sources:

• Event ID 4769: A Kerberos service ticket was requested (Successful detection).

• Event ID 5136: Directory Service Object Modified (Failed/Missing detection).

  1. 2.

    2. Scenario: Detecting the TGS Request (The "Roast")

Objective: Detect the request for a Service Ticket (TGS) against a user account that is not a standard service account. Logic: We monitor Event ID 4769 (Kerberos Service Ticket Operations). The critical indicators are:

  1. Target Service: A standard user account (sasuke_uchiha99) rather than a machine ($) or SQL/IIS service.

  1. Encryption Type: 0x17 (RC4-HMAC). Attackers often force RC4 because it is significantly easier to crack offline than AES (0x12 or 0x11).

KQL Query: (Based on the successful detection in your screenshot)

Event
| where EventLog == "Security"
| where EventID == 4769
| extend ServiceName = extract(@"Service Name:\s+(.+)", 1, RenderedDescription)
| extend TargetUserName = extract(@"Target User Name:\s+(.+)", 1, RenderedDescription)
| extend EncryptionType = extract(@"Ticket Options:\s+Encryption Type:\s+(.+)", 1, RenderedDescription)
| where ServiceName has "sasuke_uchiha99" // The victim user
| where RenderedDescription has "0x17" // RC4 Encryption (Weak)
| project TimeGenerated, TargetUserName, ServiceName, EncryptionType, Computer

Analysis:

  • Timestamp: 2026-02-10 11:28:03 PM

  • Event: A TGS ticket was requested for sasuke_uchiha99.

  1. 3.

    3. Defensive Gap Analysis (Missing Telemetry)

Observation: While we successfully detected the exploitation (Ticket Request), we failed to detect the setup (SPN Injection).

  • Missing Event: Event ID 5136 (Directory Service Object Modified).

  • Root Cause: The auditing policy for "Directory Service Changes" might be enabled, but the specific SACL (System Access Control List) on the User Objects (or the specific OU) likely does not audit "Write Property" for the servicePrincipalName attribute.

  • Risk: Without Event 5136, we cannot see who made Sasuke roastable (Attribution to tanjiro_kamado21 is lost), only that Sasuke was roasted.

  • Remediation: Configure a SACL on the "Users" OU to audit Write events for "All Extended Rights" or specifically the servicePrincipalName attribute.
