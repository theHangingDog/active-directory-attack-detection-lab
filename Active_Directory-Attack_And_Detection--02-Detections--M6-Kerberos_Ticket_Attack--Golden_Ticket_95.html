Golden Ticket


Detection & Telemetry (Golden Ticket Persistence)

  1. 1.

    1. Overview

Attack Vector: The attacker utilized a compromised account to perform a DCSync attack, exfiltrating the krbtgt account's NTLM hash. This secret material was then used to forge a Golden Ticket (Ticket Granting Ticket) for the itachi_anbu01 identity. This forged ticket was injected into memory to provide permanent, high-privileged access to the Domain Controller (DC01), allowing for administrative remote command execution.

Key Log Sources:

• Security Event ID 4662: Directory Service Object Access (Detects the prerequisite DCSync to steal the krbtgt hash).

• Sysmon Event ID 1: Process Creation (Captures the local execution and presence of the forgery tool, mimikatz.exe).

• Security Event ID 4624: Successful Logon (Detects the forged session's authentication to the DC).

  1. 2.

    2. Scenario: Detecting the Prerequisite (krbtgt Credential Theft)

Objective: Detect the extraction of the krbtgt account credentials via unauthorized replication requests.

Logic: In a DCSync attack, the attacker mimics a Domain Controller to request account secrets. We monitor Event ID 4662 on the DC for "Control Access" operations against the domain object involving replication GUIDs, specifically when initiated by a non-machine account.

KQL Query:

Analysis:

  • Subject: gojo_satoru69.

  • Pattern: The telemetry captures a massive burst of 144 items at approximately 5:03 PM. This high-volume activity by a standard user account is a definitive signature of a DCSync attack used to obtain the krbtgt hash.

  1. 3.

    3. Scenario: Detecting Forgery Tool Execution (Mimikatz)

Objective: Detect the presence and execution of the Mimikatz binary used to forge and inject the Golden Ticket. Logic: We monitor Sysmon Event ID 1 for any process creation events where the image or command line explicitly references mimikatz.exe, regardless of the user context.

KQL Query:

Analysis:

  • Execution Evidence: The logs capture mimikatz.exe being executed multiple times from C:\Users\itachi_anbu01\Downloads\.

  • Company Metadata: Even if the file was renamed, the company field "gentilkiwi (Benjamin DELPY)" serves as a high-fidelity indicator of the tool's true identity.

  1. 4.

    4. Scenario: Identifying Pass-the-Ticket Anomalies

Objective: Identify forgeries by spotting Kerberos tickets with impossible lifetimes.

Logic: Standard Kerberos TGTs typically expire within 10 hours. Mimikatz-forged Golden Tickets default to a 10-year lifespan.

Analysis:

  • Artifact: The offensive execution window confirms the forged ticket for itachi_anbu01 has a lifetime starting 2/22/2026 and ending 2/20/2036.

  • Detection Strategy: While primarily a memory artifact, Blue Teams can hunt for this by correlating Event ID 4624 (Network Logon) for high-privileged accounts originating from workstations where those users do not have a legitimate interactive presence.

  1. 5.

    5. Defensive Recommendations (Mitigation)

  1. Reset krbtgt Password Twice: This is the only method to invalidate existing Golden Tickets, as it forces the rotation of the master signing key.

  1. Tiered Administration: Ensure Domain Admin accounts are restricted from logging into anything other than Tier 0 assets (Domain Controllers) to prevent credential theft.

  1. Harden Account Monitoring: Apply a System Access Control List (SACL) to the krbtgt account to alert on any access attempts not originating from the DC's SYSTEM context.

  1. Endpoint Protection: Implement AppLocker or Windows Defender Application Control (WDAC) to prevent the execution of unauthorized binaries like Mimikatz.
