SAM and Registry Hive


Credential Dumping (SAM and Registry Hive)

  1. 1.

    1. Hypothesis & Concept

Hypothesis: If an attacker gains local Administrator privileges on a Windows endpoint (e.g., WRS01), they can access the SAM (Security Account Manager) database located in the Registry.

Concept: The SAM database stores local user passwords in NTLM hash format. These hashes are encrypted using a key stored in the SYSTEM registry hive. To dump the credentials, we need read access to both the HKLM\SAM and HKLM\SYSTEM hives.

  • Goal: Extract the NTLM hash for the built-in Administrator (RID 500) to perform Pass-the-Hash attacks or crack it offline.

  1. 2.

    2. Technique 1: Living off the Land (Reg Save & Impacket)

Objective: Extract the registry hives using native Windows commands (reg.exe) to evade antivirus that might flag tools like Mimikatz, and then decrypt them offline.

Step 1: Save Registry Hives (Windows)

Tool: reg.exe

Command:

reg save HKLM\SAM C:\temp\sam

reg save HKLM\SYSTEM C:\temp\system

Findings: The commands successfully exported the registry hives to C:\temp. This method is often stealthy because reg.exe is a trusted Microsoft binary.

Step 2: Offline Decryption (Kali)

Objective: Use the System Boot Key to decrypt the SAM database and retrieve hashes.

Tool: impacket-secretsdump

Command:

secretsdump.py -sam sam -system system LOCAL

Findings: Impacket successfully retrieved the Boot Key (0x3609...) and dumped the local hashes. We recovered the NTLM hash for Administrator (RID 500) and gojo_satoru69 (RID 1000).

  1. 3.

    3. Technique 2: Local Memory Execution (Mimikatz)

Objective: Dump the SAM database directly from memory (LSASS) without touching the disk.

Tool: mimikatz.exe

Command:

privilege::

debug

token

::

elevate

lsadump

::

sam

Findings:

  • token::elevate successfully impersonated NT AUTHORITY\SYSTEM.

  • lsadump::sam parsed the registry and output the SysKey, SAMKey, and the NTLM hash for the Administrator account: 10b3977c....

  1. 4.

    4. Technique 3: Post-Exploitation Automation (LaZagne)

Objective: Use an automated tool to recover passwords from not just the SAM, but also browsers, Wi-Fi profiles, and other local secrets.

Tool: LaZagne.exe

Command:

.\LaZagne.exe all

Findings: LaZagne decrypted the system MasterKey and successfully dumped the SAM hashes (Hashdump passwords) and LSA Secrets. This confirms multiple avenues for credential theft are available once local admin is achieved.

  1. 5.

    5. Technique 4: Remote Dumping (CrackMapExec)

Objective: Remotely dump local credentials using the SMB protocol, simulating lateral movement.

Tool: CrackMapExec (NetExec)

Command:

crackmapexec smb 192.168.56.30 -u

'gojo_satoru69'

-p

'Infinity@SixEyes'

 --sam

Findings: Using the valid domain credentials of gojo_satoru69 (who is a local admin on WRS01), we successfully dumped the SAM database remotely. The tool extracted 4 hashes, including the local Administrator and Guest accounts.

  1. 6.

    6. Impact Statement

Dumping the SAM database allows an attacker to:

  1. Lateral Movement: If the local Administrator password is reused across machines (a common misconfiguration), the attacker can compromise every machine in the network using Pass-the-Hash.

  1. Persistence: The attacker can crack the local admin password and use it to re-enter the machine even if domain credentials are changed.
