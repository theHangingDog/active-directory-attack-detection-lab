SAM and Registry Hive


Detection & Telemetry (SAM and Registry Hive Dumping)

  1. 1.

    1. Overview

Attack Vector: The attacker utilized multiple techniques to extract local password hashes from the Security Account Manager (SAM) database on the workstation (WRS01). This included "Living off the Land" using the native reg.exe utility to export registry hives, executing known credential dumping tools like Mimikatz and LaZagne directly on the host, and performing remote dumping via CrackMapExec over SMB.

Key Log Sources:

  • Sysmon Event ID 1: Process Creation (Captures the exact command lines for reg.exe, mimikatz.exe, and LaZagne.exe).

  • Sysmon Event ID 11: File Creation (Can detect the creation of sam.save or similar output files).

  • Security Event ID 7045: Service Installation (Associated with remote tools like CrackMapExec).

  1. 2.

    2. Scenario: Detecting Registry Hive Export (Reg.exe)

Objective: Detect the use of the native Windows registry tool (reg.exe) to manually export the SAM and SYSTEM hives. This technique is often used to evade antivirus signature detection.

Logic: We monitor endpoint telemetry for the execution of reg.exe with the save argument, specifically targeting the HKLM\SAM and HKLM\SYSTEM keys.

KQL Query: 

Sysmon_Events
| where EventID == 1
| where CommandLine has "reg" 
| where CommandLine has "save"
| where CommandLine has_any ("HKLM\\SAM", "HKLM\\SYSTEM")
| project TimeGenerated, UserName, Computer, CommandLine, ParentCommandLine

Analysis:

  • Command: The logs clearly capture the attacker running "C:\Windows\system32\reg.exe" save HKLM\SAM C:\temp\sam and the corresponding command for the SYSTEM hive.

  • Context: Legitimate administrators rarely export these specific registry hives to a temporary directory (C:\temp) using the command line. This pattern is a high-fidelity indicator of credential theft preparation.

  1. 3.

    3. Scenario: Detecting Known Offensive Tools (Mimikatz & LaZagne)

Objective: Detect the execution of known credential dumping binaries on the endpoint.

Logic: Attackers often drop tools like Mimikatz or LaZagne to automate the dumping process. We monitor Sysmon Event ID 1 for these specific image names or their unique command-line arguments.

KQL Query:

Sysmon_Events
| where EventID == 1
| where Image has_any ("mimikatz.exe", "LaZagne.exe") 
   or CommandLine has_any ("lsadump::sam", "sekurlsa::logonpasswords", "all") 
| project TimeGenerated, User, Computer, Image, CommandLine, ParentImage

Analysis:

  • LaZagne: The logs capture the execution of LaZagne.exe with the argument all, which instructs the tool to dump secrets from browsers, Wi-Fi, and the SAM.

  • Mimikatz: The logs show the execution of mimikatz.exe. Even if the attacker renames the binary, the command line arguments (e.g., privilege::debug) would often remain detectable.

  • Conclusion: The presence of these tools in the Downloads folder indicates a lack of AppLocker/WDAC enforcement on the workstation.

  1. 4.

    4. Scenario: Detecting Remote Dumping (CrackMapExec/NetExec)

Objective: Detect credential dumping initiated from a remote attacker machine over SMB. Logic: Tools like CrackMapExec often use the Service Control Manager to execute commands on the target. They create a temporary service with a randomized name to run the dump command. We look for System Event ID 7045 (A service was installed) where the service name is random or suspicious.

KQL Query:

Event
| where EventID == 7045
| where Source == "Service Control Manager"
| where RenderedDescription has_any ("PSEXESVC", "BTOBTO") // Or look for random strings
| project TimeGenerated, Computer, RenderedDescription

Analysis:

  • Indicator: If CrackMapExec was used with the --sam flag, it likely created a temporary service to execute the registry save or retrieval command. Detecting this service creation correlates the network-based attack to the endpoint activity.

  1. 5.

    5. Defensive Recommendations (Mitigation)

  1. Attack Surface Reduction (ASR): Enable the ASR rule "Block credential stealing from the Windows local security authority subsystem (lsass.exe)" to prevent tools like Mimikatz from reading LSASS memory.

  1. Restrict Local Admin Rights: Ensure standard users do not have local Administrator privileges. Without these rights, they cannot read the SAM or SYSTEM hives.

  1. Implement LAPS: Use the Local Administrator Password Solution (LAPS) to randomize the local Administrator password on every machine. This ensures that even if the SAM is dumped, the retrieved hash cannot be used to move laterally to other machines.

  1. Monitor Registry Access: Configure SACLs on HKLM\SAM and HKLM\SYSTEM to generate Event ID 4663 whenever a non-system process attempts to read these keys.
