BloodHound


Detection Goal: Identify suspicious Active Directory reconnaissance consistent with BloodHound collectors (SharpHound, BloodHound‑Python).

Key Indicators:

  • Large LDAP queries (Event ID 1644) from a single source IP, often requesting thousands of objects.
  • Kerberos TGS requests (Event ID 4769) targeting many services in a short time.
  • Unusual enumeration patterns (e.g., shortest path queries, excessive group membership lookups).

Detection Strategy:

Monitor Directory Service logs for Event ID 1644 and Security logs for Event ID 4769, focusing on query volume, source IP concentration, and honeytoken activity.

KQL Query (Event Table)

Event
| where Source == "Microsoft-Windows-Security-Auditing"
| where EventID in (1644, 4769)   // Large LDAP queries & Kerberos TGS requests
| extend ParsedData = parse_json(EventData)
| extend TargetUser = tostring(ParsedData["TargetUserName"]),
         SourceIp   = tostring(ParsedData["IpAddress"]),
         Records    = toint(ParsedData["RecordsReturned"])
| summarize Count = count(), MaxRecords = max(Records)
          by SourceIp, TargetUser, bin(TimeGenerated, 10m)
| where Count > 20 or MaxRecords > 1000 or TargetUser == "svc_backup"
| project TimeGenerated, SourceIp, TargetUser, Count, MaxRecords
| order by TimeGenerated desc

What this does

  • Filters for Event ID 1644 (LDAP queries) and 4769 (Kerberos TGS requests).
  • Extracts TargetUserName, SourceIp, and query size.
  • Aggregates by 10‑minute windows to catch bursts of activity.
  • Flags when query counts or record sizes exceed thresholds (common in BloodHound runs).
