Domain Cache Credential


Credential Dumping (Domain Cached Credentials / DCC2)

  1. 1.

    1. Hypothesis & Concept

Hypothesis: To allow users to log into their workstations when a Domain Controller is unreachable (e.g., a laptop disconnected from the corporate network), Windows caches their domain credentials locally. If we have local Administrator privileges, we can extract these cached hashes.

Concept:

• Storage: Domain Cached Credentials (DCC) are stored in the Registry under HKLM\SECURITY\Cache.

• Encryption: The hashes are protected by the LSA Secret key (NL$KM), which is encrypted by the system boot key located in the SYSTEM hive.

• Hash Type: Modern Windows systems use MS-Cache v2 (DCC2). Unlike NTLM hashes, DCC2 hashes are heavily salted (using the username as the salt) and use PBKDF2 with thousands of iterations.

• Crucial Limitation: DCC2 hashes cannot be used for Pass-the-Hash (PtH) attacks. They must be cracked offline to recover the plaintext password.

  1. 2.

    2. Technique 1: Living off the Land (Reg Save & Impacket)

Objective: Extract the necessary registry hives natively to avoid triggering endpoint defenses, and then perform the complex decryption process offline on an attacker-controlled machine.

Step 1: Save Registry Hives (Windows)

Tool: reg.exe

Command:

reg save HKLM\SYSTEM C:\tmp\SYSTEM

reg save HKLM\SECURITY C:\tmp\SECURITY

Findings: The commands successfully exported the SYSTEM and SECURITY registry hives to the C:\tmp directory. These files contain the encrypted DCC2 hashes and the decryption keys required to unlock them.

Step 2: Offline Decryption (Kali)

Objective: Use Impacket to parse the exported hives, decrypt the NL$KM LSA secret, and extract the DCC2 hashes.

Tool: impacket-secretsdump

Command:

impacket-secretsdump -security SECURITY -system SYSTEM LOCAL

Findings: secretsdump.py successfully retrieved the Boot Key (0x3609...) and decrypted the cached domain logon information. We successfully extracted the DCC2 hashes for multiple high-value targets who had previously accessed this workstation, including tanjiro_kamado21, gojo_satoru69, svc_saiyan, and naruto_uzumaki07.

  1. 3.

    3. Technique 2: Local Memory & Registry Execution (Mimikatz)

Objective: Automate the extraction of cached credentials directly on the compromised host using Mimikatz.

Tool: mimikatz.exe

Commands:

privilege::

debug

token

::

elevate

lsadump

::

cache

Findings:

token::elevate successfully elevated our context to NT AUTHORITY\SYSTEM.

  • The lsadump::cache command automatically located the SysKey, decrypted the NL$KM key, and enumerated the MsCacheV2 hashes for all cached users.

  • The output provided 8 cached profiles (Iteration 10240), displaying the username and the corresponding DCC2 hash for offline cracking.

  1. 4.

    4. Impact Statement & Next Steps

Extracting Domain Cached Credentials exposes the passwords of any domain user who has physically or interactively logged into the compromised machine.

Next Step (Weaponization):

Because DCC2 cannot be passed, the extracted hashes must be cracked. The standard approach utilizes Hashcat with mode 2100:

hashcat -m 2100 dcc2_hashes.txt wordlist.txt

If a highly privileged user (like a Domain Admin or Helpdesk account) previously logged into WRS01 and has a weak password, cracking their DCC2 hash yields their plaintext password, leading directly to broader domain compromise.
